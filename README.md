# 52deck
Deck of cards game
Class Project

Epic: Interactive Playing Card Deck (HTML/CSS/Vanilla JavaScript)

Goal

Build a browser-based interactive deck of playing cards using only HTML, CSS, and vanilla JavaScript. The application should render a full deck visually and provide a foundation for future interactive gameplay and animations.

User Story 1 — Render a Standard Deck of Cards

As a user, I want to see a complete deck of playing cards rendered in the browser so that I can visually interact with the deck.

Acceptance Criteria

A standard 52-card deck is generated dynamically with JavaScript

Each card displays:

Suit
Rank
Basic visual styling
Cards are rendered inside a dedicated game/table container

Red suits display in red styling

Black suits display in black styling

The layout is responsive across desktop and mobile screen sizes

User Story 2 — Create Reusable Card Components

As a developer, I want cards to use reusable HTML/CSS structures so that styling and interactivity remain maintainable.

Acceptance Criteria

Card markup follows a consistent structure
CSS classes are modular and reusable
Card sizing is controlled through CSS variables or centralized styles
Styling supports future states:
Face up
Face down
Selected
Hovered
Disabled
Prepare for Future JavaScript Enhancements

As a developer, I want the project structured cleanly so that advanced interactivity can be added later.

Acceptance Criteria

JavaScript is separated into logical functions/modules
Card data is stored in structured objects
Event listeners are centralized where practical
CSS architecture supports animation expansion
No external libraries or frameworks are used
